% ABSTRACT
\begin{abstract}
This study evaluates the predictive performance of traditional econometric and machine learning models in forecasting equity returns within the Philippine Stock Exchange, with a specific focus on Bank of the Philippine Islands (BPI). Using daily data from 2019 to 2026, the study constructs a comprehensive feature set incorporating technical indicators, cross-asset signals (PSEi and BDO returns), and macroeconomic variables such as the USD/PHP exchange rate and the U.S. 10-year Treasury yield. Three prediction horizons, namely one-day, three-day, and five-day cumulative returns, are modeled under a strict chronological 80/20 train-test split to preserve temporal integrity. The framework compares a baseline Ordinary Least Squares (OLS) model against multiple machine learning algorithms, including Random Forest, SVM, XGBoost, and ANN, with hyperparameters optimized rolling-origin cross-validation. Results show that while tree-based ensembles achieve strong in-sample performance, they suffer from overfitting and weaker out-of-sample generalization. In contrast, regularized models such as SVM and ANN demonstrate more stable and robust predictive accuracy. Additionally, performance improves at longer horizons, with five-day forecasts outperforming shorter-term predictions due to reduced noise. These findings support the use of hybrid and regularized machine learning approaches for modeling return dynamics in frontier markets.
\end{abstract}

% SECTION 1: INTRODUCTION
\section{Introduction}
\label{sec:intro}
% ASSIGNED TO: Jehan


The Philippine Stock Exchange (PSE) has proven to be inefficient in terms of adaptivity and shifting volatility regimes \parencite{ahmed2022adaptive}. Consequently, daily stock return prediction has remained a considerable challenge in the PSE. Considering this, static linear assumptions are structurally incapable of capturing regime-dependent non-linear dynamics that determine price formation in the Philippine market. Thus, presenting a critical gap in knowledge in terms of return predictability for the PSE. Especially for systemically important equities within the banking sector.

An increasing amount of empirical literature has investigated the use of machine learning techniques in ASEAN equity markets, finding significant gains in forecasting performance over standard econometric models. According to prior studies on the ASEAN-5 economies, it has been demonstrated that algorithms like Random Forest, Support Vector Machines, and ensemble boosting methodologies may predict stock index movements with accuracy with some models, such as AdaBoost, commonly perform better than traditional methods (Shaik et al., 2024). Similarly, Jayasuriya (2019) discussed that data from cross-national ASEAN assessments shows that machine learning frameworks, especially Artificial Neural Networks, outperform linear regression models in terms of predicting accuracy, with reported performance levels surpassing 90% in certain instances. Additionally, further research using Southeast Asian datasets demonstrates the adaptability of non-linear approaches in emerging market contexts by providing evidence that even relatively straightforward machine learning models, like Support Vector Machines, are capable of outperforming conventional statistical techniques when forecasting market metrics (Chaengkham & Wianwiwat, 2021).

Beyond ASEAN markets, foregoing studies on frontier economies provide support to the utilization of machine learning in circumstances marked by increased volatility, lower liquidity, and structural inefficiencies. According to Chowdhurry et al. (2020), machine learning methods are better suited to capturing intricate, non-linear patterns in such data sets, while traditional financial models frequently fail because of missing or unstable parameters. Akter et al. (2022) claims that the difficulties of extremely volatile frontier market data have led to the development of sophisticated architectures, such as stacking ensembles and deep learning models, which show better forecast accuracy than both traditional econometric and independent machine learning models. Cross-regional studies from various frontier and emerging countries, particularly Africa, demonstrates that machine learning algorithms may correctly forecast financial crises, volatility regimes, and return dynamics by combining macroeconomic and sentiment-based factors (Naeem et al., 2024). However, despite these developments, Bustos et al. (2025) further highlights limitations such as overfitting, unpredictable out-of-sample performance, and uncertainty across market regimes, indicating that model design and data structure continue to have a significant impact on prediction success. These findings highlight the necessity for hybrid and context-specific modeling frameworks by establishing machine learning as a highly effective yet flawed technique in frontier market forecasting.


Nevertheless, prior studies conducted in this field have two persistent shortcomings. First, a majority of studies only target single forecast horizons. This disregards how model performance differs or degrades across multi-day windows. Second, macroeconomic factors, specifically global liquidity proxies such as the U.S. 10-year Treasury Yield and the USD/PHP exchange rate, are often omitted from the study rather than used as structural predictors. Thus, limiting the external validity and practicality of existing models for market practitioners. Local equity valuations are deterministic outputs of global liquidity conditions. The U.S. 10-Year Treasury Yield serves as the "global gravity" for all risky assets. Rising yields in developed markets trigger capital outflows from frontier regions. The USD/PHP exchange rate captures this currency-channel risk. It also tracks the impact of remittance-driven liquidity on domestic consumption. These variables are the non-negotiable anchors of any predictive framework.


In this paper, we will utilize machine learning to forecast short-term equity returns for BDO and BPI. We deploy four machine learning algorithms to bridge this analytical gap. These include Random Forest, Support Vector Machines, XGBoost, and Artificial Neural Networks. The primary objective is to forecast returns across one-day, three-day, and five-day horizons.  These particular institutions were chosen because financial services dominate the PSEi weighting. BPI and BDO are the undisputed leaders in domestic credit intermediation. Moreover, these entities generate the largest share of systematic risk in the Philippine economy \parencite{godoy2025measuring}. Lastly, institutions function as the primary transmission mechanism for domestic monetary policy. Their combined market capitalization ensures high liquidity and systemic relevance. They represent the most robust proxy for Philippine macroeconomic health, as their price action aggregates both retail sentiment and institutional capital flows.


% SECTION 2: LITERATURE REVIEW
\section{Literature Review}
\label{sec:litreview}

\subsection{The Adaptive Market Hypothesis in Frontier Markets}
Unconditional weak-form efficiency systematically fails to describe equity pricing in frontier markets. Empirical evidence from six ASEAN economies reveals that market inefficiency predictably spikes during systemic macroeconomic crises \parencite{huynh2025composite}. Consequently, the Adaptive Market Hypothesis provides a more precise framework for understanding cyclical predictability driven by shifting volatility regimes \parencite{ahmed2022adaptive}. This adaptive inefficiency manifests within the Philippine Stock Exchange (PSE) as distinct, exploitable patterns of momentum and mean reversion. Dynamic factor modeling of the PSE confirms that latent macroeconomic trends persistently drive predictable out-of-sample price movements \parencite{lim2026dynamic}. Frontier market predictability ultimately stems from delayed institutional information processing and reactive retail liquidity dynamics.

\subsection{Econometric Limitations and Machine Learning Efficacy}
Ordinary Least Squares (OLS) regressions structurally fail to map high-dimensional, multicollinear financial data. A canonical comparative analysis demonstrates that tree-based ensembles and neural networks definitively outperform traditional linear models \parencite{gu2020empirical}. These advanced architectures optimize predictive accuracy by computationally capturing deep non-linear interactions between momentum, localized liquidity, and historical volatility. Tree-based ensembles actively mitigate the curse of dimensionality without imposing rigid, pre-specified additive constraints. An equal-weighted ensemble of these algorithms successfully isolates highly profitable out-of-sample statistical arbitrage signals \parencite{krauss2017deep}. Machine learning models thus replace manual feature engineering with automated, data-driven mathematical abstraction.

\subsection{Sequential Memory and the Ensemble Approach}
Deep learning architectures are uniquely required to extract chronological sequence dependencies from financial time series. Long Short-Term Memory (LSTM) networks consistently outperform memory-free classification models by explicitly retaining long-term structural dependencies \parencite{fischer2018deep}. These memory gates allow LSTMs to isolate optimal trading signals driven by high localized volatility and short-term reversal profiles. However, the application of highly parameterized neural networks must contend with the No Free Lunch theorem of mathematical optimization. Machine learning architectures capturing deep non-linear patterns frequently fail to translate statistical predictability into profitable trading strategies \parencite{bustos2025machine}. Therefore, hybrid ensemble methodologies remain essential to hedge against shifting macroeconomic distributions and algorithmic vulnerabilities \parencite{waldow2021machine}.

\subsection{Macroeconomic Drivers of Philippine Banking Equities}
Philippine banking equities function as deterministic outputs of highly specific, systemic macroeconomic variables. The domestic banking sector consistently generates the absolute largest share of aggregate systemic risk within the Philippine economy \parencite{godoy2025measuring}. This extreme vulnerability is directly driven by exogenous macroeconomic shocks to domestic output, policy rates, and currency valuations. Furthermore, domestic currency depreciation substantially alters economic liquidity and fundamentally impacts corporate balance sheet risk profiles \parencite{tancangco2024effects}. For systemically critical entities like BDO and BPI, valuations remain highly sensitive to net open foreign exchange positions. Any robust predictive model must mathematically capture these non-linear macroeconomic transmission mechanisms alongside traditional pricing indicators.

\subsection{Multi-Horizon Predictability and Signal Decay}
The mathematical forecasting horizon fundamentally dictates the underlying signal-to-noise ratio of equity return predictions. One-day forecasts are overwhelmed by microstructure noise, restricting high-frequency statistical arbitrage strictly to transient mean reversion. Conversely, extending forecasts to five-day horizons successfully smooths execution noise to expose persistent, structural macroeconomic trends. This extension introduces the mathematical phenomenon of horizon decay, severely degrading predictive accuracy due to compounding exogenous uncertainty. Consequently, hybrid predictive frameworks must deploy multi-horizon architectures to simultaneously capture short-term reversals and medium-term momentum channels. Maximizing risk-adjusted returns requires precise temporal alignment between the chosen algorithmic architecture and the specific prediction target.

Raw data for the domestic series were exported from Investing.com in CSV format at daily frequency, while the global macro baseline was retrieved via a financial data API. Data ingestion and alignment are managed using the \texttt{here} library to ensure path portability and reproducible processing. Each series contains the closing price, intraday high, low, and open, trading volume, and the daily percentage change. Because Philippine market holidays and the global macro/FX series do not share identical non-trading calendars, the merged dataset requires structural alignment and imputation prior to feature construction.

\subsection{Data Preprocessing and Standardization}

To ensure consistency across the diverse data sources, a unified cleaning pipeline is implemented in R using the \texttt{tidyverse} ecosystem. The preprocessing function standardizes all column headers to a lowercase, snake-case format and parses date strings using the ISO-standard MDY format. Numeric fields, frequently exported with comma separators, are stripped of non-numeric characters before conversion to double-precision floats. Volume strings containing "M" (millions) or "K" (thousands) suffixes are parsed and scaled by their respective factors ($10^6$ and $10^3$) to maintain a uniform integer scale. Inclusion criteria mandated complete daily records for the primary equities, while any outlier artifacts (e.g., negative prices or volumes) were programmatically screened and excluded.

A critical step in the time-series alignment involves the construction of a continuous daily calendar index. The dataset is merged onto a complete sequence of dates spanning the minimum and maximum of the sample period, ensuring that no gaps exist for weekends or market holidays. Missing values resulting from this expansion or from asynchronous reporting are addressed through last-observation-carried-forward (LOCF) imputation. This procedure preserves the most recent market information as the prevailing state until a new observation is recorded, effectively maintaining a continuous, daily-frequency series required for multi-horizon modeling.

\subsection{Target Variable Construction}

The three prediction targets are constructed as follows. The one-day ahead log return is:

\begin{equation}
    r_{t+1} = \log\left(\frac{P_{t+1}}{P_t}\right)
    \label{eq:r1}
\end{equation}

The three-day and five-day cumulative returns are:

\begin{equation}
    r_{t:t+3} = \sum_{i=1}^{3} r_{t+i}
    \label{eq:r3}
\end{equation}

\begin{equation}
    r_{t:t+5} = \sum_{i=1}^{5} r_{t+i}
    \label{eq:r5}
\end{equation}

Separate models are estimated for each target. All predictive features use only information available at time $t$ or earlier to eliminate look-ahead bias.

\subsection{Technical Indicators}

Three categories of technical indicators are constructed from the BPI closing price and log return series. All indicators use only information available at or before time $t$.

\subsubsection*{Trend Indicators}

Three simple moving averages are computed over windows of 5, 10, and 20 trading days:

\begin{equation}
    MA_k(t) = \frac{1}{k}\sum_{i=0}^{k-1} P_{t-i}, \quad k \in \{5,\, 10,\, 20\}
\end{equation}

Two exponential moving averages are computed via the standard recursive formula:

\begin{equation}
    EMA_{k,t} = \alpha_k\, P_t + (1 - \alpha_k)\, EMA_{k,t-1}, \quad \alpha_k = \frac{2}{k+1}, \quad k \in \{12,\, 26\}
\end{equation}

initialized with the simple average over the first $k$ observations. A moving-average ratio captures the cross-term relationship between short- and long-run trend:

\begin{equation}
    MA\_ratio_t = \frac{MA_5(t)}{MA_{20}(t)}
\end{equation}

Values above unity indicate that short-run price momentum exceeds its longer-run trend level, signaling a potential continuation pattern.

\subsubsection*{Momentum Indicators}

Lagged log returns over one, two, and three days are included as direct autoregressive predictors:

\begin{equation}
    r_{t-j} = \log\!\left(\frac{P_{t-j}}{P_{t-j-1}}\right), \quad j \in \{1,\, 2,\, 3\}
\end{equation}

Five-day and ten-day rolling momentum are defined as the difference between the current log return and the return $k$ periods prior:

\begin{equation}
    mom_{k,t} = r_t - r_{t-k}, \quad k \in \{5,\, 10\}
\end{equation}

The 14-day Relative Strength Index (RSI) measures the speed and magnitude of recent directional price changes:

\begin{equation}
    RSI_{14,t} = 100 - \frac{100}{1 + RS_t}, \quad RS_t = \frac{\overline{G}_{14,t}}{\overline{L}_{14,t}}
\end{equation}

where $\overline{G}_{14,t}$ and $\overline{L}_{14,t}$ are the 14-day exponential moving averages of daily gains and losses, respectively. RSI values above 70 conventionally signal overbought conditions; values below 30 signal oversold conditions.

The Moving Average Convergence Divergence (MACD) indicator and its nine-day signal line are:

\begin{equation}
    MACD_t = EMA_{12,t} - EMA_{26,t}, \qquad Signal_t = EMA_9(MACD_t)
\end{equation}

The spread between the MACD and its signal line encodes medium-frequency momentum transitions.

\subsubsection*{Volatility Indicators}

Rolling standard deviations of log returns are computed over 5-day, 10-day, and 20-day windows:

\begin{equation}
    \sigma_{k,t} = \sqrt{\frac{1}{k-1}\sum_{i=1}^{k}\left(r_{t-i} - \bar{r}_{k,t}\right)^2}, \quad k \in \{5,\, 10,\, 20\}
\end{equation}

where $\bar{r}_{k,t}$ is the rolling mean return over the same window. A volatility ratio captures the relative level of short- to medium-run realized risk:

\begin{equation}
    vol\_ratio_t = \frac{\sigma_{5,t}}{\sigma_{10,t}}
\end{equation}

Ratios above unity indicate that near-term volatility is elevated relative to its recent baseline, which can precede reversals or continuations depending on the prevailing regime.

\subsection{Cross-Asset and Market Features}

Three cross-asset return series are included to capture market-wide, sector-level, and macroeconomic information channels beyond BPI's own price history.

The contemporaneous PSEi log return captures systematic, market-wide movements on the same trading day, while a one-day lagged PSEi return is included as a predictor of next-day BPI performance:

\begin{equation}
    r^{PSEi}_t = \log\!\left(\frac{PSEi_t}{PSEi_{t-1}}\right)
\end{equation}

BDO Unibank returns serve as the cross-sectional competitor feature, reflecting sector-specific shocks that affect Philippine banking broadly but may be priced into BDO before BPI, or vice versa, due to differences in liquidity and investor base:

\begin{equation}
    r^{BDO}_t = \log\!\left(\frac{P^{BDO}_t}{P^{BDO}_{t-1}}\right)
\end{equation}

The daily log return on the USD/PHP spot exchange rate is included as the primary macroeconomic variable:

\begin{equation}
    r^{FX}_t = \log\!\left(\frac{FX_t}{FX_{t-1}}\right)
\end{equation}

An appreciation of the Philippine peso (i.e., a decline in USD/PHP) can affect bank profitability through foreign currency-denominated assets, liabilities, and remittance-driven deposit flows, making the exchange rate a theoretically motivated predictor for banking sector equity returns.

The second macroeconomic variable is the log return of the U.S. 10-Year Treasury Constant Maturity Yield (\texttt{\textasciicircum TNX}). In a frontier market, the U.S. 10-year yield acts as the global discount rate. It governs the valuation of all risky assets.

\begin{equation}
    r^{US10Y}_t = \log\!\left(\frac{Y^{US10Y}_t}{Y^{US10Y}_{t-1}}\right)
\end{equation}

Figure \ref{fig:macro_correlation} plots the cumulative log returns of four key series: the target asset (BPI), its sector competitor (BDO), the local market benchmark (PSEi), and the global macro baseline (U.S. 10-Year Yield). Fluctuations in the global risk-free rate trigger immediate revaluation of emerging market bank multiples. Rising yields historically exert downward pressure on banking valuations through currency-channel risk and tightening domestic liquidity. The inclusion of $r^{US10Y}_t$ forces the predictive models to distinguish between idiosyncratic domestic shocks and systemic shifts in global capital flows.

\subsection{Feature Transformations and Regime Variable}

\subsubsection*{Feature Transformations}

Two engineered features are included to capture nonlinear and conditional relationships among the base indicators. An interaction term between the one-day lagged return and short-run realized volatility allows the models to detect whether momentum is predictive conditional on the prevailing risk environment:

\begin{equation}
    interact_t = r_{t-1} \times \sigma_{5,t}
\end{equation}

A squared short-run volatility term is included as a direct nonlinear transformation:

\begin{equation}
    vol^2_{5,t} = \left(\sigma_{5,t}\right)^2
\end{equation}

This term captures the convex relationship between extreme volatility states and subsequent return distributions, a pattern well-documented in empirical asset pricing \parencite{gu2020empirical}.

\subsubsection*{Regime Variable}

A data-driven binary indicator identifies high-volatility regimes without imposing calendar-based date restrictions:

\begin{equation}
    HiVol_t = \mathbf{1}\!\left[\,\sigma_{20,t} > 2\,\bar{\sigma}\,\right]
\end{equation}

where $\sigma_{20,t}$ is the 20-day rolling standard deviation of BPI log returns and $\bar{\sigma}$ is the historical mean of that series computed on the training set only, ensuring that the regime threshold is not contaminated by future information. A value of one flags periods in which near-term realized volatility is at least twice the long-run baseline---capturing episodes such as the March 2020 market dislocation and subsequent macroeconomic stress periods---without hardcoding specific dates.

\subsubsection*{Look-Ahead Bias Controls}

All features described in this section are constructed strictly from information available at or before time $t$. Moving-window operators reference only historical observations; no forward-looking quantities enter any predictor. The three target variables ($r_{t+1}$, $r_{t:t+3}$, $r_{t:t+5}$) are stored as separate lead-shifted columns and are never included among the predictors. The regime threshold $\bar{\sigma}$ is estimated on the training partition and applied to the test set without re-fitting, consistent with the strict time-series split described in Section~\ref{sec:methodology}.

% SECTION 4: METHODOLOGY
\section{Methodology}
\label{sec:methodology}

This section outlines the statistical and machine learning frameworks deployed to answer the core research question: whether non-linear algorithmic architectures can capture adaptive inefficiency in Philippine banking equities better than traditional linear models. The predictive framework utilizes a hybrid approach. It integrates classical econometrics with non-linear machine learning architectures to systematically handle the high-dimensional interactions of momentum, volatility, and macroeconomic variables. The objective is to forecast multi-horizon log returns for the Philippine banking sector. All machine learning algorithms and resampling procedures were implemented programmatically in R utilizing the \texttt{tidymodels} ecosystem.

\subsection{Data Partitioning and Temporal Integrity}

Predicting financial time series requires a strict chronological split. Random shuffling is prohibited. It introduces look-ahead bias. The dataset spans from January 2, 2019, to April 1, 2026. This period comprises 1,853 daily observations. The first 80\% of the sample serves as the training set, established chronologically without shuffling. The remaining 20\% is reserved for out-of-sample testing. This split ensures the models are evaluated on unseen market regimes. It simulates a real-world trading environment. Crucially, the target variables for each specific horizon are strictly excluded from the corresponding predictor matrix, effectively nullifying any potential data leakage.

\subsection{Target Variable Construction}

The study estimates separate models for three distinct investment horizons. All targets are expressed as log returns. This transformation ensures stationarity.
\begin{enumerate}
    \item \textbf{1-day Return ($r_{t+1}$):} Defined as $\log(P_{t+1}/P_t)$. It represents short-term price discovery.
    \item \textbf{3-day Cumulative Return ($r_{t:t+3}$):} Defined as $\sum_{i=1}^{3} r_{t+i}$. It captures multi-day momentum.
    \item \textbf{5-day Cumulative Return ($r_{t:t+5}$):} Defined as $\sum_{i=1}^{5} r_{t+i}$. It reflects weekly trend persistence.
\end{enumerate}
All features are lagged at time $t$ or earlier. This eliminates data leakage.

\subsection{Model Specifications}

\subsubsection{Ordinary Least Squares (OLS) Benchmark}
The OLS model serves as the linear baseline. It assumes a constant relationship between features and returns. The specification follows $r_{t+k} = \alpha + \beta \mathbf{X}_t + \epsilon_t$. $\mathbf{X}_t$ represents the vector of technical and macro features. We utilize Newey-West standard errors. This adjustment accounts for heteroskedasticity and autocorrelation.

\subsubsection{Random Forest (RF)}
The RF model employs an ensemble of 500 decision trees. It uses bootstrap aggregation to reduce variance. The \texttt{ranger} engine implements this architecture. We tune the number of features per split (\textit{mtry}) and the minimum node size. This prevents the model from fitting to idiosyncratic market noise. Feature importance extraction relies on impurity-based metrics.

\subsubsection{Support Vector Machine (SVM)}
The SVM utilizes a Radial Basis Function (RBF) kernel. It maps features into a high-dimensional space to find non-linear hyperplanes, implemented via the \texttt{kernlab} engine. Feature standardization is mandatory. We scale all predictors to zero mean and unit variance. To systematically prevent data leakage, the scaling parameters (\texttt{step\_normalize}) are derived solely from the training set and applied exclusively within the cross-validation workflow. We tune the cost parameter ($C$) and the kernel width ($\sigma$).

\subsubsection{Gradient Boosting (XGBoost)}
XGBoost implements a sequence of weak learners via the \texttt{xgboost} engine. Each tree corrects the residuals of its predecessor. It optimizes a regularized objective function. This prevents overfitting. We tune the learning rate, tree depth, and L1/L2 regularization terms. It is the most computationally efficient architecture in the suite. Feature importance extraction relies on gain-based metrics.

\subsubsection{Artificial Neural Network (ANN)}
The ANN consists of a single-layer perceptron, utilizing the \texttt{nnet} engine. It uses a decay parameter for regularization. The architecture identifies complex interactions through hidden neurons. Similar to the SVM pipeline, predictor features are rigorously standardized exclusively on the training partition. We optimize the number of hidden units and the decay rate. This ensures the network does not memorize the training data.

\subsection{Validation and Hyperparameter Tuning}

Hyperparameter optimization utilizes a rolling origin cross-validation strategy to respect temporal precedence. We employ the \texttt{rsample::rolling\_origin()} function with an initial training window of 200 observations and an assessment window of 60 observations, approximately equivalent to one trading quarter. To manage computational overhead, we utilized a skip increment of 100 observations between folds. 

Each model was tuned using a Latin Hypercube grid search to ensure efficient coverage of the high-dimensional parameter space. The Random Forest models utilized a 15-point grid for \textit{mtry} (ranging from 3 to 13) and \textit{min\_n} (5 to 30). The XGBoost architecture employed a 20-point grid across five parameters: learning rate ($10^{-3}$ to 0.1), tree depth (3 to 8), \textit{min\_n} (5 to 30), loss reduction, and early stopping iterations (10 to 50). Support Vector Machines and Artificial Neural Networks utilized 15-point grids for their respective kernel and regularisation parameters. \subsection{Evaluation Metrics and Overfitting Assessment}

We assess model performance through statistical and financial lenses on both in-sample (Train) and out-of-sample (Test) observations to explicitly diagnose overfitting.
\begin{itemize}
    \item \textbf{Root Mean Squared Error (RMSE):} Measures the magnitude of prediction deviations.
    \item \textbf{Mean Absolute Error (MAE):} Provides a robust measure of average error.
    \item \textbf{R-squared ($R^2$):} Quantifies the proportion of variance explained.
    \item \textbf{Directional Accuracy (DA):} Calculates the frequency of correct sign predictions. This metric is the primary indicator of economic utility.
\end{itemize}
All models are compared against a naïve lagged-return benchmark. Overfitting is quantified by observing the performance delta between Training and Testing metrics. A high Training DA combined with a low Testing DA serves as the cardinal diagnostic signature of algorithmic over-parameterization.

\section{Discussion}
\label{sec:discussion}

\subsection{Model Comparison and Algorithmic Robustness}

The empirical comparison between linear econometric benchmarks and non-linear machine learning architectures reveals a complex predictive landscape that challenges the conventional wisdom that ``more parameters equals better predictions.'' While machine learning models are theoretically superior at capturing high-dimensional interactions, they remain highly susceptible to idiosyncratic noise when the signal-to-noise ratio is low. The collapse of the Random Forest model out-of-sample illustrates the dangers of epistemological overconfidence in financial modeling. As Table~\ref{tab:rf_xgboost_metrics} documents, the RF's Training $R^2$ of 0.783 at the 5-day horizon suggests a model that explains 78\% of training variance---yet its Test $R^2$ of -0.134 confirms that these explanatory powers are entirely illusory out-of-sample. The high capacity of 500 decision trees, each allowed to grow to a minimum node size of 14, allowed the architecture to memorize specific volatility wiggles from the 2019--2024 training period that did not recur in the 2025--2026 regime.

Conversely, the Support Vector Machine achieved the most robust out-of-sample performance in the suite. Its 5-day Directional Accuracy of 49.0\% (Table~\ref{tab:svm_ann_metrics}) surpasses the naive benchmark by 6.1 percentage points, and its Train-Test RMSE gap of -0.0002 is the narrowest among all models. This success is attributed to explicit mathematical regularization. The SVM cost parameter ($C = 8.23$ at 5-day, per Table~\ref{tab:best_hyperparameters}) and RBF kernel width ($\sigma = 0.00011$) smooth the decision boundary, forcing the model to ignore training set outliers and focus on the dominant directional trend. The kernel trick maps the 24-dimensional feature space into an infinite-dimensional Hilbert space, where a linear hyperplane can separate positive from negative returns---but the soft-margin formulation ensures that this separation is not achieved by memorizing individual training points.

The Artificial Neural Network achieves nearly identical performance to the SVM (Test DA of 49.2\% at 5-day), but with a fundamentally different architecture. The single-layer perceptron with 3 hidden units and L2 weight decay of 0.624 represents a heavily constrained function approximator. Its success demonstrates that in low-signal environments, architectural restraint---few parameters, strong regularization---outperforms raw capacity. This finding aligns with \textcite{gu2020empirical}, who show that neural networks with dropout and weight decay consistently outperform unregularized deep architectures in equity return prediction.

The OLS baseline, despite its linear restrictions, remains competitive with the machine learning models on directional accuracy. Its 5-day DA of 51.1\% exceeds both SVM (49.0\%) and ANN (49.2\%), though its negative $R^2$ confirms that the linear model captures direction without magnitude. The OLS model's advantage lies in its transparency: every coefficient is interpretable, every standard error is computable, and the Newey-West HAC adjustment ensures valid inference under heteroskedasticity and autocorrelation. For practitioners who require explainable predictions---regulators, risk managers, and portfolio constructors---the OLS model offers a compelling combination of directional accuracy and interpretability that black-box architectures cannot match.

\subsection{Multi-Horizon Analysis and Signal Decay}

The predictive efficacy of all models is fundamentally dictated by the forecasting horizon, and the decay pattern is both systematic and economically interpretable. At the 1-day horizon, the signal-to-noise ratio is exceptionally low: microstructure noise---bid-ask bounce, order flow imbalances, and transient retail sentiment---dominates price action. All architectures exhibited poor Directional Accuracy at this interval, with SVM (32.8\%), ANN (35.8\%), and OLS (33.1\%) all performing barely above random guessing. The Random Forest's 1-day DA of 43.7\% appears stronger but is entirely driven by in-sample memorization, as the Test $R^2$ of -0.014 confirms.

Extending the forecast to a 3-day cumulative horizon introduces a smoothing effect that exposes structural macroeconomic momentum. The DA improvements are substantial: OLS jumps from 33.1\% to 49.0\% (+15.9pp), SVM from 32.8\% to 48.9\% (+16.1pp), and ANN from 35.8\% to 47.5\% (+11.7pp). This extension allows the models to filter out high-frequency noise and capture the persistent directional drift driven by institutional capital flows. The 3-day window is approximately one-half of a trading week, sufficient to absorb the impact of scheduled macroeconomic releases (CPI, GDP, BSP policy rate decisions) that typically arrive on known calendar dates.

The 5-day horizon represents the sweet spot for directional predictability. OLS achieves 51.1\% DA---the only model to cross the 50\% threshold---while SVM reaches 49.0\% and ANN 49.2\%. The superior performance of regularized models at this horizon confirms that frontier market predictability is a low-frequency phenomenon tied to structural trend persistence rather than short-term mean reversion. Figure~\ref{fig:horizon_decay} in the Results section documents this decay pattern visually: the RMSE slope for SVM (0.0162 from 1-day to 5-day) is the shallowest, indicating that the RBF kernel's smooth decision boundary degrades most gracefully as cumulative uncertainty compounds.

The naive benchmark's own DA pattern---26.6\% (1-day), 31.6\% (3-day), 29.7\% (5-day)---reveals an important structural insight: the lagged return's predictive power peaks at the 3-day horizon and then declines. This suggests that the autocorrelation structure of BPI returns has a characteristic timescale of approximately 3 trading days, beyond which the memory of the previous day's return dissipates into the cumulative noise of intervening shocks. Models that can exploit multi-feature interactions---OLS with its 23 predictors, SVM with its kernel-mapped feature space---continue to gain predictive power beyond this 3-day inflection point, while the single-feature naive heuristic loses ground.

\subsection{Economic Interpretation of Feature Significance}

The variable importance profiles and OLS coefficient estimates jointly confirm that Philippine banking equities function as deterministic outputs of global liquidity conditions and domestic momentum dynamics. The high ranking of the U.S. 10-Year Treasury Yield ($r^{US10Y}_t$) in short-horizon models validates the global gravity hypothesis advanced in the Introduction. Rising developed market yields trigger immediate capital flight from emerging markets like the Philippines, as institutional investors rebalance portfolios toward higher-yielding, lower-risk developed market assets. This macroeconomic transmission mechanism exerts downward pressure on banking multiples through the currency channel and tightening domestic liquidity.

The OLS coefficient estimates provide a quantified map of these transmission channels. At the 1-day horizon, the dominant predictors are the lagged BPI return ($lag1$, coefficient = -0.220) and the contemporaneous BPI return ($r\_bpi$, coefficient = -0.210), both significant at the 1\% level. These negative coefficients confirm short-horizon mean reversion: a 1\% positive return today predicts a 22 basis point negative return tomorrow, holding all other features constant. This is the classic microstructure bounce---the PSE's relatively thin order book means that large institutional trades push prices temporarily away from equilibrium, and the reversion is swift. The economic magnitude is meaningful: given the BPI's daily return standard deviation of 1.63\% (Table~\ref{tab:descriptive_stats}), a one-standard-deviation positive return predicts a 3.6 basis point reversion the following day.

At the 5-day horizon, the predictive landscape shifts decisively. The FX return ($r\_fx$) achieves 1\% significance with a coefficient of -0.672, the largest macroeconomic coefficient in the entire model. The economic interpretation is direct: a one-standard-deviation peso appreciation (0.28\%, per Table~\ref{tab:descriptive_stats}) predicts approximately 19 basis points of positive BPI return over the next 5 trading days. This currency channel operates through multiple mechanisms. First, peso appreciation improves the domestic purchasing power of the Philippine banking sector's predominantly peso-denominated loan portfolios, reducing real debt burdens and improving credit quality. Second, a stronger peso signals capital inflows, which disproportionately benefit the largest and most liquid banks---BPI chief among them. Third, the Bangko Sentral ng Pilipinas typically responds to peso strength by easing monetary policy, which reduces funding costs and expands net interest margins for domestic banks.

The dominance of the MA Ratio and lagged returns across longer horizons suggests that the PSE exhibits persistent trend-following behavior. These momentum indicators capture the reactive nature of local market participants---predominantly retail investors and domestic institutions---who process institutional information with a temporal lag. The interaction term ($interact$), significant at the 3-day horizon ($t = 2.11$), indicates that the predictive power of lagged returns is amplified in high-volatility environments: when the 5-day rolling volatility is elevated, the mean-reversion signal from $lag1$ becomes stronger. This conditional relationship is economically intuitive: volatility spikes trigger panic selling, which creates overshooting that is subsequently corrected. The squared volatility term ($vol5\_sq$), significant at both 3-day ($t = -2.08$) and 5-day ($t = -2.68$) horizons with large negative coefficients, captures the convexity of this effect: extreme volatility states are followed by return compression, as the market enters a consolidation phase after the shock dissipates.

\subsection{Regime Effects and the Adaptive Market Hypothesis}

The observed decay in predictive performance across regimes provides strong empirical support for the Adaptive Market Hypothesis (AMH). The training sample (January 2019 to October 2024) was dominated by COVID-19 dislocations, the March 2020 market crash, the 2022 inflation spike, and the subsequent monetary tightening cycle---each representing a unique statistical distribution with distinct volatility, correlation, and return characteristics. The high-volatility regime dummy ($HiVol_t$) flags only 2.2\% of the full sample as extreme volatility days, but these 57 observations disproportionately shaped the tree ensembles' decision boundaries. Random Forest's 500 trees learned to associate specific feature combinations with crisis-era return patterns that simply did not recur in the test period.

The test period (October 2024 to April 2026) represents a fundamentally different macroeconomic regime. The BSP had completed its tightening cycle, inflation had moderated, and the PSEi entered a period of consolidation with lower realized volatility and narrower return dispersion. In this environment, the patterns that were statistically significant in 2021--2022---sharp mean reversions after crash-level drawdowns, volatility clustering at crisis levels, and crisis-driven cross-asset correlations---had decayed by 2025. The Random Forest's Test $R^2$ of -0.134 at the 5-day horizon is the quantitative signature of this regime mismatch: the model was calibrated for a crisis environment and deployed in a normalization environment.

The SVM and ANN, by contrast, generalize across regimes because their regularization parameters structurally limit their sensitivity to regime-specific patterns. The SVM's RBF kernel with $\sigma = 0.00011$ produces a decision boundary that is smooth enough to remain valid across both crisis and normalization regimes. The ANN's 3 hidden units and weight decay of 0.624 constrain the network to learn only the most persistent, regime-invariant relationships---primarily the directional drift driven by institutional capital flows and the currency channel. This robustness confirms that the Philippine market is not a stationary system but an evolving ecological environment where algorithmic strategies must continuously adapt to shifting participation rates, institutional mandates, and macroeconomic conditions. The AMH predicts exactly this pattern: predictability is not a fixed property of the market but a time-varying function of the adaptive pressure exerted by changing environmental conditions.

\subsection{Practical Trading Implications}

The empirical findings carry direct implications for practitioners deploying predictive models in the Philippine equity market. A Directional Accuracy of 49--51\% at the 5-day horizon may appear modest, but it is economically meaningful when combined with proper position sizing and risk management. A simple long-only strategy that enters BPI positions when the 5-day SVM predicts a positive return and exits when it predicts negative would, over the 522-observation test period, achieve approximately 256 correct directional calls versus 266 incorrect ones. While the raw count is near-even, the economic value depends critically on the magnitude of correct versus incorrect predictions: if the model correctly identifies the direction of larger-magnitude returns (which the SVM's conservative prediction range suggests it does), the strategy generates positive expected returns net of transaction costs.

The hierarchy of model robustness established in this study---SVM $\approx$ ANN $>$ OLS $>$ XGBoost $\gg$ Random Forest---provides a clear deployment guide. For directional positioning over a weekly horizon, the SVM's 49.0\% DA and near-zero overfitting gap make it the preferred architecture. For intraday or next-day hedging, the OLS model's 33.1\% DA---while barely above random---offers the advantage of full interpretability: the trader can see exactly which predictors are driving the signal and adjust position sizes accordingly. The Random Forest, despite its impressive-sounding Training $R^2$ of 0.783, should never be deployed in production: its Test $R^2$ of -0.134 guarantees that it will destroy value relative to a simple mean-return forecast.

The multi-horizon approach itself is a practical tool. Rather than selecting a single forecast horizon, practitioners should deploy models across all three horizons simultaneously and use the consensus signal. When the 1-day, 3-day, and 5-day models all predict the same direction, the combined signal is substantially stronger than any individual horizon's prediction. This ensemble-of-horizons approach hedges against the regime-specific vulnerabilities that afflict any single temporal window.

\section{Conclusion}

This study examined whether non-linear machine learning models can better capture adaptive inefficiencies in Philippine banking equities compared to traditional linear approaches. The results show that predictive performance is highly dependent on the forecast horizon. At the 1-day level, all models struggle due to noise and short-term randomness. However, as the horizon extends to 3-day and 5-day returns, more stable patterns emerge, leading to improved directional accuracy. Interestingly, higher model complexity does not guarantee better performance. Random Forest, despite strong in-sample results, fails out-of-sample due to overfitting, while more constrained models such as SVM and ANN perform more consistently. At the same time, the OLS benchmark remains competitive, even achieving the highest directional accuracy at the 5-day horizon, suggesting that simpler models can still be effective in low signal-to-noise environments.

From a practical perspective, these findings suggest that analysts should focus less onshort-term predictions and instead prioritize slightly longer horizons where signals are more reliable. Model selection should emphasize robustness rather than complexity, with regularized approaches offering a better balance between flexibility and generalization. At the same time,linear models remain valuable, especially when interpretability is important. A combined ormulti-horizon approach may further improve decision-making by reducing dependence on any single prediction window.

Despite these findings, the study has several limitations. The analysis focuses on a single stockwithin the Philippine banking sector, which may limit generalizability. The feature set, while comprehensive, does not include alternative data such as sentiment or high-frequency indicators that may contain additional predictive information. In addition, real-world trading factors such as transaction costs and liquidity constraints are not incorporated, meaning that statistical performance may not directly translate to economic profitability. Future research can address these limitations by expanding the dataset across multiple assets, incorporating adaptive or regime-switching models, and integrating alternative data sources. Evaluating actual trading strategies under realistic market conditions would also provide a more complete assessment of the models' practical value.

Overall, the results suggest that in financial prediction, more complex models are not necessarily better. Instead, approaches that are well-regularized and aligned with the structure of the data tend to perform more reliably, especially in non-stationary market environments.

## 2. Cleaning Function

To maintain consistency between BPI, BDO, and PSEi datasets, we use a single unified cleaning function. This directly addresses the conversion issues where numeric columns with commas or unit suffixes (M/K) were previously being parsed as NA.

The function performs the following:
1. Standardizes column names to lowercase and removes non-alphanumeric characters.
2. Parses dates from MDY format.
3. Cleans numeric columns (Price, Open, High, Low) by removing commas before conversion.
4. Calculates Volume (Vol) by identifying 'M' (millions) or 'K' (thousands) suffixes.
5. Adds a prefix to the cleaned columns to prevent name collisions during the merge.

```{r cleaning_functions}
#' Clean financial data exported from Investing.com
#' @param df The raw data frame to be processed.
#' @param prefix A string prefix to append to each column except the date.
#' @return A processed data frame with standardized numeric columns.
clean_investing_data <- function(df, prefix = "") {
  # Standardize all column names to be snake_case and lowercase
  names(df) <- tolower(gsub("[^a-zA-Z0-9]", "_", names(df)))
  names(df) <- gsub("_+", "_", names(df))
  names(df) <- gsub("(^_|_$)", "", names(df))
  
  df_clean <- df %>%
    mutate(
      # Standardize the date column using lubridate mdy parsing
      date = mdy(date),
      
      # Convert price-related columns to numeric by stripping commas
      across(c(price, open, high, low), ~ as.numeric(gsub(",", "", .x))),
      
      # Transform volume strings into actual numerical values
      # We check for the suffix and multiply the parsed number accordingly.
      vol = case_when(
        grepl("M", vol) ~ as.numeric(gsub("[^0-9.]", "", vol)) * 1e6,
        grepl("K", vol) ~ as.numeric(gsub("[^0-9.]", "", vol)) * 1e3,
        TRUE ~ as.numeric(gsub("[^0-9.]", "", vol))
      ),
      
      # Remove percentage signs from the change column and convert to numeric
      change_pct = as.numeric(gsub("%", "", .data[["change"]]))
    ) %>%
    # Subset to the core required columns for analysis
    select(date, price, open, high, low, vol, change_pct) %>%
    arrange(date)
  
  # Append the prefix to columns if one is provided
  if (prefix != "") {
    df_clean <- df_clean %>%
      rename_with(~ paste0(prefix, "_", .x), -date)
  }
  
  return(df_clean)
}

#' Clean Yahoo Finance yield data
#' @param df The raw data frame from tq_get.
#' @param prefix A string prefix to append to the columns.
#' @return A processed data frame with standardized yield columns.
clean_yahoo_yield_data <- function(df, prefix = "us10y") {
  df %>%
    rename(price = close) %>% # ^TNX uses 'close' as the primary price point
    select(date, price, open, high, low, volume) %>%
    rename(vol = volume) %>%
    mutate(change_pct = (price / lag(price) - 1) * 100) %>%
    rename_with(~ paste0(prefix, "_", .x), -date) %>%
    arrange(date)
}
```

## 5. Metrics Function

Produces a one-row tibble with the same 11-column schema as `04_models_rf_xgboost.Rmd`
and `05_models_svm_ann.Rmd` so `06_evaluation.Rmd` can bind all model results with
a single `bind_rows()`.

```{r metrics_fn}
compute_metrics <- function(test_actual, test_pred,
                            train_actual, train_pred,
                            naive_pred, model, horizon) {
  test_ss_res  <- sum((test_actual  - test_pred)^2,  na.rm = TRUE)
  test_ss_tot  <- sum((test_actual  - mean(test_actual,  na.rm = TRUE))^2, na.rm = TRUE)
  train_ss_res <- sum((train_actual - train_pred)^2, na.rm = TRUE)
  train_ss_tot <- sum((train_actual - mean(train_actual, na.rm = TRUE))^2, na.rm = TRUE)

  tibble(
    Model      = model,
    Horizon    = horizon,
    RMSE       = sqrt(mean((test_actual  - test_pred)^2,  na.rm = TRUE)),
    MAE        = mean(abs(test_actual   - test_pred),     na.rm = TRUE),
    R2         = 1 - test_ss_res  / test_ss_tot,
    DA         = mean(sign(test_pred)  == sign(test_actual),  na.rm = TRUE),
    Naive_DA   = mean(sign(naive_pred) == sign(test_actual),  na.rm = TRUE),
    Train_RMSE = sqrt(mean((train_actual - train_pred)^2, na.rm = TRUE)),
    Train_MAE  = mean(abs(train_actual  - train_pred),    na.rm = TRUE),
    Train_R2   = 1 - train_ss_res / train_ss_tot,
    Train_DA   = mean(sign(train_pred) == sign(train_actual), na.rm = TRUE)
  )
}
```

## 6. OLS Estimation (Three Horizons)

One model per target horizon. Naïve benchmark uses the previous BPI log return (`lag1`)
as a zero-parameter prediction for comparison. Newey-West HAC standard errors correct for
serial correlation in daily return residuals (`prewhite = FALSE` to avoid NaN SEs on
near-zero-autocorrelation series).

```{r ols_models}
ols_metrics_list   <- vector("list", length(TARGET_COLS))
naive_metrics_list <- vector("list", length(TARGET_COLS))
pred_list          <- vector("list", length(TARGET_COLS))
coef_list          <- vector("list", length(TARGET_COLS))

for (h in seq_along(TARGET_COLS)) {
  target_col    <- TARGET_COLS[h]
  horizon_label <- HORIZON_LABS[h]

  cat("\n--- OLS |", toupper(horizon_label), "horizon ---\n")

  ols_formula <- as.formula(
    paste(target_col, "~", paste(OLS_PREDICTORS, collapse = " + "))
  )

  train_h <- train_df %>% select(all_of(c(target_col, OLS_PREDICTORS)))
  test_h  <- test_df  %>% select(all_of(c(target_col, OLS_PREDICTORS)))

  # Fit OLS on training data only
  ols_fit <- lm(ols_formula, data = train_h)
  cat("Fit:", n_train, "obs |", length(coef(ols_fit)) - 1L, "predictors\n")

  # Newey-West HAC variance-covariance matrix
  nw_vcov <- NeweyWest(ols_fit, lag = NW_LAG, prewhite = FALSE)
  nw_test <- coeftest(ols_fit, vcov = nw_vcov)

  # Extract coefficient table — broom::tidy() supports coeftest objects
  coef_tidy <- tidy(nw_test) %>%
    rename(std_error = std.error, t_stat = statistic, p_value = p.value) %>%
    mutate(
      stars   = case_when(
        p_value < 0.01 ~ "***",
        p_value < 0.05 ~ "**",
        p_value < 0.10 ~ "*",
        TRUE           ~ ""
      ),
      horizon = horizon_label
    )
  coef_list[[h]] <- coef_tidy

  # Predictions
  train_pred <- predict(ols_fit, newdata = train_h)
  test_pred  <- predict(ols_fit, newdata = test_h)
  naive_pred <- test_df$lag1          # previous BPI log return
  naive_train_pred <- train_df$lag1

  # OLS metrics
  ols_metrics_list[[h]] <- compute_metrics(
    test_actual  = test_h[[target_col]],
    test_pred    = test_pred,
    train_actual = train_h[[target_col]],
    train_pred   = train_pred,
    naive_pred   = naive_pred,
    model        = "OLS",
    horizon      = horizon_label
  )

  # Naive benchmark metrics
  naive_metrics_list[[h]] <- compute_metrics(
    test_actual  = test_h[[target_col]],
    test_pred    = naive_pred,
    train_actual = train_h[[target_col]],
    train_pred   = naive_train_pred,
    naive_pred   = naive_pred,
    model        = "Naive",
    horizon      = horizon_label
  )

  cat("OLS   | RMSE:", round(ols_metrics_list[[h]]$RMSE,  5),
      "| DA:", round(ols_metrics_list[[h]]$DA,   4),
      "| R2:", round(ols_metrics_list[[h]]$R2,   4), "\n")
  cat("Naive | RMSE:", round(naive_metrics_list[[h]]$RMSE, 5),
      "| DA:", round(naive_metrics_list[[h]]$DA,  4), "\n")

  # Store predictions for CSV export
  pred_df <- tibble(
    date                            = test_df$date,
    !!paste0("actual_", target_col) := test_h[[target_col]],
    !!paste0("ols_pred_", target_col) := test_pred
  )
  pred_list[[h]] <- pred_df
}

# Combine all metrics

naive_metrics   <- bind_rows(naive_metrics_list)
all_ols_metrics <- bind_rows(ols_metrics, naive_metrics)
```

## 4. Metrics Helper

Computes RMSE, MAE, R² (out-of-sample), Directional Accuracy, and Naïve Directional Accuracy for one model-horizon evaluation. The naïve benchmark is the previous day's BPI log return (`lag1`) as a stand-in prediction.

```{r metrics_fn}
compute_metrics <- function(test_actual, test_pred, train_actual, train_pred, naive_pred, model, horizon) {
  test_ss_res   <- sum((test_actual - test_pred)^2)
  test_ss_total <- sum((test_actual - mean(test_actual))^2)
  train_ss_res  <- sum((train_actual - train_pred)^2)
  train_ss_total <- sum((train_actual - mean(train_actual))^2)

  tibble(
    Model      = model,
    Horizon    = horizon,
    RMSE       = sqrt(mean((test_actual - test_pred)^2)),
    MAE        = mean(abs(test_actual - test_pred)),
    R2         = 1 - test_ss_res / test_ss_total,
    DA         = mean(sign(test_pred) == sign(test_actual)),
    Naive_DA   = mean(sign(naive_pred) == sign(test_actual)),
    Train_RMSE = sqrt(mean((train_actual - train_pred)^2)),
    Train_MAE  = mean(abs(train_actual - train_pred)),
    Train_R2   = 1 - train_ss_res / train_ss_total,
    Train_DA   = mean(sign(train_pred) == sign(train_actual))
  )
}
```

## 5. Random Forest

Tree-based ensembles are scale-invariant; no feature normalisation is applied. We tune two hyperparameters:

- **`mtry`** (features sampled per split): controls decorrelation between trees.
- **`min_n`** (minimum leaf size): regularises tree depth.

All 500 trees use impurity-based importance scoring (`importance = "impurity"`) for feature importance extraction via `vip`.

```{r rf_models}
set.seed(RANDOM_SEED)

rf_metrics_list <- vector("list", length(TARGET_COLS))

for (h in seq_along(TARGET_COLS)) {

  target_col    <- TARGET_COLS[h]
  horizon_label <- HORIZON_LABS[h]

  cat("\nRF |", toupper(horizon_label), "horizon\n")

  # Select only the outcome + predictors to keep other target columns out of the recipe
  train_h <- train_df %>% select(all_of(c(target_col, PREDICTOR_COLS)))
  test_h  <- test_df  %>% select(all_of(c(target_col, PREDICTOR_COLS)))

  # Rolling-origin CV on training set only — no test data ever enters the fold
  cv_h <- rolling_origin(
    data       = train_h,
    initial    = CV_INITIAL,
    assess     = CV_ASSESS,
    skip       = CV_SKIP,
    cumulative = TRUE
  )
  cat("CV slices:", nrow(cv_h), "\n")

  # No normalisation needed — trees are scale-invariant
  rf_recipe <- recipe(
    as.formula(paste(target_col, "~ .")),
    data = train_h
  )

  rf_spec <- rand_forest(
    mtry  = tune(),
    min_n = tune(),
    trees = RF_TREES
  ) %>%
    set_engine(
      "ranger",
      importance = "impurity",   # required for vip() to extract importance scores
      seed       = RANDOM_SEED
    ) %>%
    set_mode("regression")

  rf_wf <- workflow() %>%
    add_recipe(rf_recipe) %>%
    add_model(rf_spec)

  rf_grid <- grid_latin_hypercube(
    mtry(range  = c(RF_MTRY_MIN, RF_MTRY_MAX)),
    min_n(range = c(RF_MINN_MIN, RF_MINN_MAX)),
    size = RF_GRID_SIZE
  )

  rf_tune <- tune_grid(
    rf_wf,
    resamples = cv_h,
    grid      = rf_grid,
    metrics   = metric_set(rmse, mae, rsq),
    control   = control_grid(verbose = TRUE, save_pred = FALSE)
  )

  best_rf <- select_best(rf_tune, metric = "rmse")
  cat("Best mtry:", best_rf$mtry, "| Best min_n:", best_rf$min_n, "\n")

  final_rf_wf  <- finalize_workflow(rf_wf, best_rf)
  final_rf_fit <- fit(final_rf_wf, data = train_h)

  rf_preds       <- predict(final_rf_fit, new_data = test_h)$.pred
  rf_train_preds <- predict(final_rf_fit, new_data = train_h)$.pred
  naive_preds    <- test_df$lag1   # previous day's return as naive benchmark

  rf_metrics_list[[h]] <- compute_metrics(
    test_actual  = test_h[[target_col]],
    test_pred    = rf_preds,
    train_actual = train_h[[target_col]],
    train_pred   = rf_train_preds,
    naive_pred   = naive_preds,
    model        = "Random Forest",
    horizon      = horizon_label
  )
  cat("RMSE:", round(rf_metrics_list[[h]]$RMSE, 6),
      "| DA:",  round(rf_metrics_list[[h]]$DA,   4),
      "| Naive DA:", round(rf_metrics_list[[h]]$Naive_DA, 4), "\n")

  # 1. Feature Importance Plot
  vip_plot <- final_rf_fit %>%
    extract_fit_parsnip() %>%
    vip(num_features = 15, aesthetics = list(fill = "#141F52", alpha = 0.9)) +
    labs(
      title    = "Random Forest — Feature Importance (Top 15)",
      subtitle = paste0(toupper(horizon_label), " Horizon | Test RMSE: ", round(rf_metrics_list[[h]]$RMSE, 6)),
      caption  = "Source: Custom Machine Learning Pipeline. Out-of-sample testing on 20% holdout.",
      x = "Impurity-Based Importance",
      y = NULL
    ) +
    theme_quant()

  ggsave(
    here("paper", "figures", paste0("rf_importance_", horizon_label, ".png")),
    vip_plot,
    width = 8, height = 6, dpi = 300
  )
  
  # 2. Predicted vs Actual Plot
  pred_vs_actual_plot <- ggplot(tibble(Actual = test_h[[target_col]], Predicted = rf_preds), aes(x = Actual, y = Predicted)) +
    geom_point(color = "#141F52", alpha = 0.6) +
    geom_abline(intercept = 0, slope = 1, color = "#E3120B", linetype = "dashed", linewidth = 1) +
    labs(
      title    = "Random Forest — Predicted vs Actual Returns",
      subtitle = paste0(toupper(horizon_label), " Horizon | Test RMSE: ", round(rf_metrics_list[[h]]$RMSE, 6)),
      caption  = "Source: Custom Machine Learning Pipeline. Out-of-sample testing on 20% holdout.",
      x = "Actual Log Return",
      y = "Predicted Log Return"
    ) +
    theme_quant()

  ggsave(
    here("paper", "figures", paste0("rf_pred_actual_", horizon_label, ".png")),
    pred_vs_actual_plot,
    width = 8, height = 6, dpi = 300
  )

  saveRDS(
    final_rf_fit,
    here("output", "models", paste0("rf_", horizon_label, ".rds")),

  cat("Outputs saved for RF", horizon_label, "\n")
}
```

## 4. Metrics Helper

```{r metrics_fn}
compute_metrics <- function(test_actual, test_pred, train_actual, train_pred, naive_pred, model, horizon) {
  test_ss_res   <- sum((test_actual - test_pred)^2)
  test_ss_total <- sum((test_actual - mean(test_actual))^2)
  train_ss_res  <- sum((train_actual - train_pred)^2)
  train_ss_total <- sum((train_actual - mean(train_actual))^2)

  tibble(
    Model      = model,
    Horizon    = horizon,
    RMSE       = sqrt(mean((test_actual - test_pred)^2)),
    MAE        = mean(abs(test_actual - test_pred)),
    R2         = 1 - test_ss_res / test_ss_total,
    DA         = mean(sign(test_pred) == sign(test_actual)),
    Naive_DA   = mean(sign(naive_pred) == sign(test_actual)),
    Train_RMSE = sqrt(mean((train_actual - train_pred)^2)),
    Train_MAE  = mean(abs(train_actual - train_pred)),
    Train_R2   = 1 - train_ss_res / train_ss_total,
    Train_DA   = mean(sign(train_pred) == sign(train_actual))
  )
}
```

## 5. Support Vector Machine (RBF Kernel)

SVM requires all numeric features to be on a comparable scale. The recipe includes `step_normalize(all_numeric_predictors())`, which centres and scales each predictor to zero mean and unit variance. Critically, `prep()` is called once per CV fold inside the tidymodels workflow — the normalisation parameters are derived only from the training portion of each fold and are never exposed to the assessment or test data.

We tune two kernel hyperparameters:

| Parameter | Controls |
|---|---|
| `cost` | Regularisation; lower values = wider margin, more slack |
| `rbf_sigma` | RBF kernel bandwidth; higher values = tighter decision boundary |

```{r svm_models}
set.seed(RANDOM_SEED)

svm_metrics_list <- vector("list", length(TARGET_COLS))

for (h in seq_along(TARGET_COLS)) {

  target_col    <- TARGET_COLS[h]
  horizon_label <- HORIZON_LABS[h]

  cat("\nSVM |", toupper(horizon_label), "horizon\n")

  train_h <- train_df %>% select(all_of(c(target_col, PREDICTOR_COLS)))
  test_h  <- test_df  %>% select(all_of(c(target_col, PREDICTOR_COLS)))

  cv_h <- rolling_origin(
    data       = train_h,
    initial    = CV_INITIAL,
    assess     = CV_ASSESS,
    skip       = CV_SKIP,
    cumulative = TRUE
  )
  cat("CV slices:", nrow(cv_h), "\n")

  # step_normalize fits mean and SD on the training partition of each CV fold only.
  # Fitting the scaler on the full training set before CV would leak fold-assessment info.
  svm_recipe <- recipe(
    as.formula(paste(target_col, "~ .")),
    data = train_h
  ) %>%
    step_normalize(all_numeric_predictors())

  svm_spec <- svm_rbf(
    cost      = tune(),
    rbf_sigma = tune()
  ) %>%
    set_engine("kernlab") %>%
    set_mode("regression")

  svm_wf <- workflow() %>%
    add_recipe(svm_recipe) %>%
    add_model(svm_spec)

  svm_grid <- grid_latin_hypercube(
    cost(),
    rbf_sigma(),
    size = SVM_GRID_SIZE
  )

  svm_tune <- tune_grid(
    svm_wf,
    resamples = cv_h,
    grid      = svm_grid,
    metrics   = metric_set(rmse, mae, rsq),
    control   = control_grid(verbose = TRUE, save_pred = FALSE)
  )

  best_svm <- select_best(svm_tune, metric = "rmse")
  cat("Best cost:", round(best_svm$cost, 5),
      "| rbf_sigma:", round(best_svm$rbf_sigma, 6), "\n")

  final_svm_wf  <- finalize_workflow(svm_wf, best_svm)
  final_svm_fit <- fit(final_svm_wf, data = train_h)

  svm_preds       <- predict(final_svm_fit, new_data = test_h)$.pred
  svm_train_preds <- predict(final_svm_fit, new_data = train_h)$.pred
  naive_preds     <- test_df$lag1

  svm_metrics_list[[h]] <- compute_metrics(
    test_actual  = test_h[[target_col]],
    test_pred    = svm_preds,
    train_actual = train_h[[target_col]],
    train_pred   = svm_train_preds,
    naive_pred   = naive_preds,
    model        = "SVM",
    horizon      = horizon_label
  )
  cat("RMSE:", round(svm_metrics_list[[h]]$RMSE, 6),
      "| DA:",  round(svm_metrics_list[[h]]$DA,   4),
      "| Naive DA:", round(svm_metrics_list[[h]]$Naive_DA, 4), "\n")

  pred_vs_actual_plot <- ggplot(tibble(Actual = test_h[[target_col]], Predicted = svm_preds), aes(x = Actual, y = Predicted)) +
    geom_point(color = "#141F52", alpha = 0.6) +
    geom_abline(intercept = 0, slope = 1, color = "#E3120B", linetype = "dashed", linewidth = 1) +
    labs(
      title    = "SVM — Predicted vs Actual Returns",
      subtitle = paste0(toupper(horizon_label), " Horizon | Test RMSE: ", round(svm_metrics_list[[h]]$RMSE, 6)),
      caption  = "Source: Custom Machine Learning Pipeline. Out-of-sample testing on 20% holdout.",
      x = "Actual Log Return",
      y = "Predicted Log Return"
    ) +
    theme_quant()

  ggsave(
    here("paper", "figures", paste0("svm_pred_actual_", horizon_label, ".png")),
    pred_vs_actual_plot,
    width = 8, height = 6, dpi = 300
  )
  cat("Scatter plot saved: paper/figures/svm_pred_actual_", horizon_label, ".png\n", sep = "")

  saveRDS(
    final_svm_fit,
    here("output", "models", paste0("svm_", horizon_label, ".rds"))
  )
  cat("Model saved: output/models/svm_", horizon_label, ".rds\n", sep = "")
}
```

## 6. Artificial Neural Network (nnet — Single Hidden Layer)

The ANN uses `nnet` via tidymodels' `mlp()` interface. Architecture: one hidden layer with a tunable number of hidden units; linear activation on the output node (regression mode). Feature normalisation is applied via the same `step_normalize()` recipe as SVM, ensuring stable gradient descent and comparable input magnitudes.

We tune two parameters:

| Parameter | Controls |
|---|---|
| `hidden_units` | Width of the single hidden layer |
| `penalty` | L2 weight decay; prevents overfitting |

**Architectural note:** `nnet` uses logistic (sigmoid) activation on hidden units and does not natively support ReLU activation or dropout. The `penalty` (weight decay) parameter provides regularisation analogous to dropout by penalising large weight magnitudes.

```{r best_model}
  best_model_name == "Random Forest" ~ "rf",
  best_model_name == "XGBoost"       ~ "xgboost",
  best_model_name == "SVM"           ~ "svm",
  best_model_name == "ANN"           ~ "ann",

)

# Load the best model and generate test predictions
best_model_path <- here("output", "models", paste0(model_file_prefix, "_1d.rds"))

if (file.exists(best_model_path)) {
  best_fit <- readRDS(best_model_path)

  # Load features for test set reconstruction
  features <- read_csv(here("data", "processed", "features.csv"), show_col_types = FALSE)
  n_total  <- nrow(features)
  n_train  <- floor(0.80 * n_total)

  # Determine predictor columns based on model type
  if (best_model_name == "OLS") {

  test_h <- test_df %>% select(all_of(c("target_1d", pred_cols)))

  if (best_model_name == "OLS") {
    test_pred <- predict(best_fit, newdata = test_h)
  } else {
```
