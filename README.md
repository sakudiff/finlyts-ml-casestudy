## Abstract

This project develops and compares econometric and machine learning models to
predict multi-horizon stock returns of two major Philippine banks: Bank of the
Philippine Islands (BPI) and BDO Unibank (BDO). Five model families are
evaluated across three prediction horizons using both statistical and financial
performance metrics.

---

## Research Question

Can machine learning models outperform a linear econometric benchmark in
predicting multi-horizon stock returns in a frontier market context (Philippine
Stock Exchange)?

---

## Models

| Model | Type | R Package |
|---|---|---|
| OLS | Benchmark econometric | `stats::lm` |
| Random Forest | Ensemble ML | `ranger` via `tidymodels` |
| Support Vector Machine | Kernel ML | `kernlab` via `tidymodels` |
| XGBoost | Gradient boosting | `xgboost` via `tidymodels` |
| ANN | Neural network | `nnet` via `tidymodels` |

---

## Prediction Horizons

| Horizon | Target |
|---|---|
| 1-day | $r_{t+1} = \log(P_{t+1} / P_t)$ |
| 3-day cumulative | $r_{t:t+3} = \sum_{i=1}^{3} r_{t+i}$ |
| 5-day cumulative | $r_{t:t+5} = \sum_{i=1}^{5} r_{t+i}$ |

Each model is estimated separately per horizon. No look-ahead bias.

## Reproducing the Analysis

All scripts are RMarkdown files in `R/`. Run sequentially from a clean R session:

| Step | Script | Purpose |
|---|---|---|
| 01 | `01_data_pull.Rmd` | Download raw data from Yahoo Finance |
| 02 | `02_feature_engineering.Rmd` | Build features, targets, regime dummy |
| 03 | `03_ols_baseline.Rmd` | Fit OLS benchmark for all horizons |
| 04 | `04_models_rf_xgboost.Rmd` | Train and tune Random Forest + XGBoost |
| 05 | `05_models_svm_ann.Rmd` | Train and tune SVM + ANN |
| 06 | `06_evaluation.Rmd` | Compute metrics, generate tables/figures |

**Reproduction command (per script):**
```r
rmarkdown::render("R/01_data_pull.Rmd")
```

Or knit directly in RStudio. All output tables and figures are written to
`paper/tables/` and `paper/figures/`.

## Methodological Notes

1. **Time-series split only.** No shuffling. Rolling-origin validation via `rsample::rolling_origin()`.
2. **No look-ahead bias.** Features use `dplyr::lag()`; targets use `dplyr::lead()` as separate columns.
3. **SVM standardization.** Scaler fitted on training set only.
4. **Hyperparameter tuning.** `tune::tune_grid()` with rolling-origin resampling, not k-fold CV.
5. **Citations.** APA 7 with DOI or URL.
